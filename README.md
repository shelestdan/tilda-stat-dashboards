# Tilda Statistics Dashboards

Статический набор дашбордов для просмотра экспортов Tilda.

Сейчас в проекте есть:

- страница входа
- главная страница-каталог дашбордов
- дашборд `RVZ`
- дашборд `VGS2000`

## Структура

- `index.html` — страница входа
- `hub.html` — главная страница со списком дашбордов
- `rvz.html` — дашборд RVZ
- `vgs2000.html` — дашборд VGS2000
- `styles.css` — общие стили
- `app.js` — логика дашбордов
- `home.js` — логика главной страницы
- `auth.js` — логика входа и расшифровки данных
- `auth-config.js` — конфигурация входа, собранная локально
- `data/` — защищенные датасеты
- `scripts/` — парсеры и сборщики данных из экспортов Tilda

## Локальный запуск

Можно открыть `index.html` напрямую или поднять простой сервер:

```bash
python3 -m http.server 4173
```

После этого проект будет доступен по адресу:

```text
http://127.0.0.1:4173
```

## GitHub Pages

Проект рассчитан на публикацию как статический сайт через GitHub Pages.
Для публичной публикации данные в `data/` хранятся в зашифрованном виде и
расшифровываются в браузере после ввода логина и пароля.

После пуша в GitHub достаточно:

1. открыть настройки репозитория
2. зайти в раздел `Pages`
3. выбрать публикацию из ветки `main`
4. указать папку `/ (root)`

Тогда вход будет открываться через `index.html`, а после авторизации будут доступны:

- `/hub.html`
- `/rvz.html`
- `/vgs2000.html`

## Как добавить новый дашборд

1. Подготовить отдельный HTML-файл дашборда.
2. Сгенерировать для него защищенный датасет.
3. Добавить запись в `DASHBOARD_REGISTRY` в `home.js`.

## Сборка защищенных payload'ов

Перед публикацией нужно локально собрать `auth-config.js` и зашифрованные датасеты:

```bash
TILDA_DASHBOARD_USER=viewer \
TILDA_DASHBOARD_PASSWORD=your-password \
./.venv/bin/python scripts/build_protected_payloads.py
```

Для нескольких пользователей можно передать JSON-файл со списком логинов и паролей:

```json
[
  { "username": "user-1", "password": "secret-1" },
  { "username": "user-2", "password": "secret-2" }
]
```

```bash
TILDA_DASHBOARD_USERS_FILE=/absolute/path/to/users.json \
./.venv/bin/python scripts/build_protected_payloads.py
```
